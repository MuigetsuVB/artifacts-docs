Equipment and items are central to character progression. This page covers all item types, equipment slots, conditions for equipping, and the equip/unequip actions.

## Overview

### Item Types

Every item in the game belongs to one of these types:

| Type | Description |
|---|---|
| `weapon` | Melee or ranged weapons that provide elemental attacks. |
| `shield` | Defensive off-hand equipment. |
| `helmet` | Head armor. |
| `body_armor` | Torso armor. |
| `leg_armor` | Leg armor. |
| `boots` | Foot armor. |
| `ring` | Accessory worn in ring slots (2 slots available). |
| `amulet` | Neck accessory. |
| `artifact` | Special equipment (3 slots, each must be a different artifact). |
| `utility` | Consumables automatically used in combat (2 slots, each must be different). |
| `rune` | Special combat ability item. |
| `bag` | Increases inventory capacity. |
| `resource` | Gathered or crafted materials used for crafting. |
| `consumable` | Items used manually outside combat (potions, food, etc.). |
| `currency` | Special currency items (e.g., task coins). |

You can browse all items using the [items endpoint](https://api.artifactsmmo.com/docs/#/operations/get_all_items_items_get) or on the [encyclopedia](https://artifactsmmo.com/encyclopedia/items).

### Item Schema

```http title="ItemSchema"
{
  "name": "Iron Sword",
  "code": "iron_sword",
  "level": 10,
  "type": "weapon",
  "subtype": "weapon",
  "description": "A sturdy iron sword.",
  "tradeable": true,
  "conditions": [
    { "code": "level", "operator": "gt", "value": 9 }
  ],
  "effects": [
    { "code": "attack_fire", "value": 24, "description": "+24 Attack Fire" },
    { "code": "attack_earth", "value": 24, "description": "+24 Attack Earth" }
  ],
  "craft": {
    "skill": "weaponcrafting",
    "level": 10,
    "items": [
      { "code": "iron", "quantity": 6 }
    ],
    "quantity": 1
  }
}
```

## Equipment Slots

Your character has 16 equipment slots:

| Slot | Accepts | Notes |
|---|---|---|
| `weapon` | Weapons | Primary damage source. |
| `shield` | Shields | Adds defensive stats. |
| `helmet` | Helmets | Head protection. |
| `body_armor` | Body armor | Main armor piece. |
| `leg_armor` | Leg armor | Lower body protection. |
| `boots` | Boots | Foot protection, may add haste. |
| `ring1` | Rings | First ring slot. |
| `ring2` | Rings | Second ring slot. |
| `amulet` | Amulets | Neck accessory. |
| `artifact1` | Artifacts | First artifact slot. |
| `artifact2` | Artifacts | Second artifact (must differ from artifact1). |
| `artifact3` | Artifacts | Third artifact (must differ from artifact1 and artifact2). |
| `utility1` | Utilities | First utility slot (quantity 1–100). |
| `utility2` | Utilities | Second utility (must differ from utility1, quantity 1–100). |
| `bag` | Bags | Increases inventory max items. |
| `rune` | Runes | Grants a combat ability. |

:::note
**Artifact slots** must each contain a different artifact. Similarly, **utility slots** must each contain a different utility item.
:::

## Equip Conditions

Items may have conditions that must be met before equipping. Conditions use the `ConditionSchema`:

```http title="ConditionSchema"
{
  "code": "level",
  "operator": "gt",
  "value": 9
}
```

| Operator | Meaning |
|---|---|
| `eq` | Equal to |
| `ne` | Not equal to |
| `gt` | Greater than |
| `lt` | Less than |
| `cost` | Costs a resource to equip |
| `has_item` | Must have item in inventory |
| `achievement_unlocked` | Must have unlocked an achievement |

The `code` field can reference any character stat (e.g., `level`, `mining_level`, `attack_fire`).

## Utilities

Utilities are special items that activate automatically during combat when specific conditions are met. They are consumed when triggered.

| Keyword | Effect |
|---|---|
| Restore | Heals X HP when the player has lost 50% of their life. |
| Splash Restore | Restores X HP at the start of the turn to the ally who has lost the most health (if below 50% HP). |
| Boost | Gives X (HP, % damage, % res, prospecting, critical strike) at the start of fight. |
| Antipoison | At the beginning of the turn, if the character has at least one poison, removes X poison damage. |

[View all utilities](https://artifactsmmo.com/encyclopedia/items?type=utility)

## Runes

Runes grant special combat abilities. You can obtain them from NPCs (e.g., NPC at position 6,13 on the overworld).

| Keyword | Effect |
|---|---|
| Burn | On first turn, applies a burn effect of X% of your attack (all elements). Damage is applied each turn and decreases by 10% each time. |
| Lifesteal | Restores X% of the total attack (all elements) in HP after a critical strike. |
| Healing | Every 3 played turns, restores X% of HP at the start of the turn. |
| Healing Aura | Every 2 played turns, heals all allies for X% of their max HP. Does not heal the caster. |
| Vampiric Strike | After a critical hit, heals the ally with the lowest HP for X% of damage dealt. Once every 3 turns. Does not heal the caster. |
| Frenzy | On critical hit, grants X% damage to self and allies until end of next turn. |
| Shell | When below 40% HP, gains X% resistance (all elements) for 3 turns. Once per combat. Boss fights only. |
| Guard | If an ally is below 50% HP, protects the lowest-HP-% ally by redirecting X% of their damage to this character until the start of next turn. Max 3 times per combat. |

[View all runes](https://artifactsmmo.com/encyclopedia/items?type=rune)

## API Endpoints

### Equip Item

`POST /my/{name}/action/equip` — Equip an item on your character.

Your character must meet all the item's conditions. The item must be in your inventory.

<Tabs>
  <TabItem label="cURL">
  ```http
  curl --request POST \
  --url https://api.artifactsmmo.com/my/{name}/action/equip \
  --header 'Accept: application/json' \
  --header 'Authorization: Bearer YOUR_TOKEN' \
  --header 'Content-Type: application/json' \
  --data '{
  "code": "iron_sword",
  "slot": "weapon"
}'
  ```
  </TabItem>
  <TabItem label="JavaScript" icon="seti:javascript">
  ```js
  const url = 'https://api.artifactsmmo.com/my/{name}/action/equip';
const options = {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    Accept: 'application/json',
    Authorization: 'Bearer YOUR_TOKEN'
  },
  body: JSON.stringify({ code: "iron_sword", slot: "weapon" })
};

try {
  const response = await fetch(url, options);
  const data = await response.json();
  console.log(data);
} catch (error) {
  console.error(error);
}
  ```
  </TabItem>
  <TabItem label="Python" icon="seti:python">
  ```python
  import requests

url = "https://api.artifactsmmo.com/my/{name}/action/equip"
headers = {
    "Accept": "application/json",
    "Content-Type": "application/json",
    "Authorization": "Bearer YOUR_TOKEN"
}
data = {"code": "iron_sword", "slot": "weapon"}

response = requests.post(url, json=data, headers=headers)
print(response.json())
  ```
  </TabItem>
  <TabItem label="Python SDK" icon="seti:python">
  ```python
  from artifacts import ArtifactsClient

with ArtifactsClient(token="YOUR_TOKEN") as client:
    char = client.character("{name}")
    char.equipment.equip(code="iron_sword", slot="weapon")
  ```
  </TabItem>
</Tabs>

| Field | Description |
|---|---|
| `code` | The item code to equip. |
| `slot` | Equipment slot: `weapon`, `shield`, `helmet`, `body_armor`, `leg_armor`, `boots`, `ring1`, `ring2`, `amulet`, `artifact1`, `artifact2`, `artifact3`, `utility1`, `utility2`, `bag`, `rune`. |
| `quantity` | Item quantity (optional, default 1). Only for utilities (1–100). |

[View full endpoint details](https://api.artifactsmmo.com/docs/#/operations/action_equip_item_my__name__action_equip_post)

### Unequip Item

`POST /my/{name}/action/unequip` — Remove an item from an equipment slot. The item returns to your inventory.

:::caution
You must have at least **1 free inventory slot** to unequip. Your character must also have enough HP to survive any stat changes from unequipping.
:::
